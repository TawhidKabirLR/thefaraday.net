What We Build

Full Stack Systems

Building complete software systems from backend to frontend, with a focus on performance, scalability, and maintainability. Every component is designed to work together as a cohesive whole, not assembled from disconnected parts.

Embedded Engineering

Designing low-level systems that interact directly with hardware, prioritizing efficiency and reliability. We write the firmware and drivers that make hardware respond exactly as intended, with no wasted cycles and no abstraction overhead where it matters.

AI & Automation

Creating intelligent systems and automation tools that integrate into real-world workflows and improve operational efficiency. We build systems that make decisions based on real data, with well-defined behavior and observable outputs.

Selected Work

Systems we've shipped.

01

Automotive

CAN Bus Gateway Controller

A multi-protocol automotive gateway built around the STM32H7, bridging CAN FD, LIN, and Ethernet buses. Developed firmware in bare-metal C with FreeRTOS scheduling, achieving sub-100µs latency across all protocol bridges while meeting ISO 26262 ASIL-B requirements.

02

Industrial IoT

Cellular IoT Sensor Platform

A low-power wireless sensor platform built on the Nordic nRF9160 for LTE-M connectivity and integrated GPS tracking. Designed for a 3-year battery life on a single cell in hostile industrial environments. The full software stack — from Zephyr RTOS to cloud ingestion — was built in-house.

03

Edge Computing

Edge AI Inference Board

A custom system-on-module with a dedicated NPU for on-device machine learning inference. Designed to process MIPI camera input and run TensorFlow Lite models under a strict 2W power budget. Custom BSP, camera ISP tuning, and inference runtime were all developed internally.

How We Work

First principles.
Every time.

01

Built from First Principles

We don't start with a framework and work backward. Every system is designed from its fundamental requirements, which means no inherited complexity and no unnecessary dependencies. We understand every component we ship.

02

Performance at Every Layer

Performance is not an optimization pass at the end. It is a design constraint from the start. We consider compute budgets, memory layouts, and latency requirements before the first line of code is written.

03

Precision Over Excess

Smaller systems are easier to reason about, test, and maintain. We resist the impulse to add abstraction where directness will do, and we prefer systems that do one thing correctly over systems that attempt everything adequately.
